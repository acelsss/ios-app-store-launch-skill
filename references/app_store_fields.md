# App Store Connect Fields

Use this reference when drafting `app_store_metadata.md`.

## Common Fields

- App name: localized product page name. Keep concise.
- Subtitle: short value proposition under the app name.
- Promotional text: short updateable marketing copy.
- Keywords: comma-separated search terms. Avoid repeating words already in the app name when space is tight.
- Description: explain core value first, then features, use cases, and tips.
- What's New: version-specific release note.
- Support URL: required for each localization.
- Privacy Policy URL: required for apps that collect data and commonly required for review readiness.
- App Review notes: explain login needs, test flow, permissions, and special setup.
- Categories: choose based on primary user intent, not implementation detail.

## Drafting Rules

- Do not mention exact paid price in the description.
- Keep copy aligned with actual app behavior and screenshots.
- Avoid unsupported claims such as "best", "guaranteed", or "100% accurate".
- For localized metadata, preserve the meaning rather than translating word-for-word.
- Add an "Open Items" section for anything the user must confirm.
